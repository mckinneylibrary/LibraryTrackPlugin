# LibraryTrack

A Koha plugin for logging reference interactions and capturing **Stories of Impact** — the moments that show a library's value. It gives front-desk staff a fast way to record day-to-day reference work and gives administrators reports and exports that turn that activity into evidence.

LibraryTrack is a standalone product. It is **not** part of the Library Staff Scheduling System that also lives in this monorepo — different purpose, branding, and database tables.

## Features

- **Log Interaction** — record reference interactions with configurable pickers (Type, Duration, Asked By, Format, Location), a question/answer pair, space-separated tags (with most-used tags promoted as quick chips), and time/date/initials. Supports "Save" and "Save & add another", with a paginated, editable list of recent entries.
- **Stories of Impact** — capture richer narratives with an outcome, patron type, program/service, staff, date, tags, and an optional photo (via image URL). Browse as a card gallery with full-story view and filters by tag, patron type, and date range.
- **Reports** — pick a date range and see summary counts by Type, Format, Location, Duration, and Patron Type, with combinable filters. Export to **CSV** and **Excel**.
- **Settings** (admin only) — rename any field label and add / remove / reorder the dropdown options for each field. Stored as a single field-configuration record.

## Architecture

The frontend is a single React + Vite app with one data layer (`src/lib/api.ts`) that runs in two modes, switched by the `VITE_KOHA_MODE` build flag:

| Mode | Persistence | Used for |
| --- | --- | --- |
| Dev / preview (`VITE_KOHA_MODE` unset) | Browser `localStorage` | Local development and the Replit preview, where no Koha database is available |
| Koha (`VITE_KOHA_MODE=true`) | Koha plugin REST via `run.pl` | The installed plugin, talking to the Perl backend |

The single-`api.ts`, two-backends design is the core principle: page components never know which backend they are talking to.

### Koha backend (Perl)

`Koha::Plugin::Com::LibTrack::LibraryTrack` (`Koha/Plugin/Com/LibTrack/LibraryTrack.pm`):

- Serves the bundled React app and a JSON API through `method=tool` (`tool.tt` bootstraps the bundle).
- Resolves the current staff user and their role for admin gating.
- Mutations are tunneled as `GET` requests carrying `_method` and a URL-safe base64 (`_body_b64`) body, because some hosted Koha setups reject intranet `POST`s and reject standard-base64 `/` characters in the URL. Handlers are written to be idempotent.

Tables created on install (all prefixed `koha_plugin_libtrack_`):

- `koha_plugin_libtrack_interactions` — reference interactions
- `koha_plugin_libtrack_stories` — Stories of Impact (photos stored as a plain `photo_url`)
- `koha_plugin_libtrack_config` — field labels, dropdown options, and promoted tags (one JSON blob)

`upgrade` re-runs install and applies a best-effort migration that adds the `photo_url` column to older installs.

## Requirements

- Koha **22.05** or newer.
- Plugins enabled in Koha (`<enable_plugins>1</enable_plugins>` in `koha-conf.xml`).

## Installation

1. Download or build the plugin package (`LibraryTrack-<date>.kpz`, see below).
2. In Koha, go to **Administration → Manage plugins → Upload plugin**.
3. Upload the `.kpz` file. Koha runs `install`, creating the plugin tables.
4. Open the plugin from the plugins list (**Run tool**) to launch LibraryTrack.

To update, upload a newer `.kpz`; Koha runs `upgrade` automatically.

## Building the package

From the repository root (or this directory):

```bash
node koha-plugin-librarytrack/build-kpz.js
```

This:

1. Builds the React frontend in Koha mode (`pnpm --filter @workspace/librarytrack run build:koha`).
2. Copies the bundle into the plugin's `htdocs/dist`.
3. Zips the `Koha/` tree into `LibraryTrack-<YYYYMMDD>.kpz`.

Packaging uses Node's `adm-zip` (resolved from the workspace root) — there is no `zip` binary in this environment. Pass an optional output directory as the first argument: `node build-kpz.js /some/dir`.

To syntax-check the Perl backend:

```bash
perl -c koha-plugin-librarytrack/Koha/Plugin/Com/LibTrack/LibraryTrack.pm
```

## Layout

```
koha-plugin-librarytrack/
├── build-kpz.js                              # Packages the .kpz (Node adm-zip)
└── Koha/Plugin/Com/LibTrack/
    ├── LibraryTrack.pm                       # Plugin backend: install/upgrade, API, routing
    └── LibraryTrack/
        ├── tool.tt                           # Bootstraps the React bundle
        └── htdocs/dist/                      # Built React app (generated by build-kpz.js)
```

The React source lives separately in `artifacts/librarytrack/` within the monorepo.
